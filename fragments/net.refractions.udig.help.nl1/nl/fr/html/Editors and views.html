Les éditeurs et les vues

Avant de commencer les tutoriels Workbench dans cette section, il faut se familiariser avec les composants du Workbench. Un Workbench se compose de :

  • Views
  • Editors

Une vue est une composante visuelle du Workbench que l'on utilise pour naviguer une hiérarchie d'informations (telles que les ressources dans le Workbench), lancer un éditeur, ou montrer les propriétés pour l'éditeur actif. Les changements effectués dans une vue sont immédiatement enregistrés. Normalement, un seul exemple d'une vue en particulier ne peut exister dans une fenêtre Workbench à la fois.
Un éditeur est également une composante visuelle. On l'utilise surtout pour éditer ou naviguer une ressource. Les changements effectués dans un éditeur se conforme au modèle Ouvrir - Enregistrer - Fermer. Plusieurs exemples d'éditeurs peuvent être ouverts dans une fenêtre Workbench à la fois.

Les vues et les éditeurs ont quelques fonctions en commun. On utilise le terme « partie » pour désigner une vue ou un éditeur. Les parties peuvent être actives ou inactives, mais une seule partie à la fois peut être active.

La partie active est identifiée par la barre de titre mise en évidence, et c'est là où les opérations telles que couper, copier et coller sont souvent effectuées. De plus, la partie active détermine le contenu de la barre d'état. Une languette d'éditeur blanche indique que l'éditeur est inactif; cependant, les vues peuvent montrer les informations d'après le dernier éditeur actif.

Si on clique sur la vue Catalogue, la barre de titre devient bleue, ce qui indique que c'est maintenant la vue catalogue – et non la vue Projets – qui est active.
