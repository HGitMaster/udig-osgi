Register Wizards

Identifier:
net.refractions.udig.registry.ui.registerWizards

Since:
0.1.0

Description:
This extention point is used to provide the registery with additional wizards allowing the discovery of new content. The wizards are presumed to call back to Catalog with a addEntry( RegistryEntry ) when the user selects finish.

Configuration Markup:

<!ELEMENT extension (wizard)*>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>

This extension point is used to list data source registration wizard extensions. Register wizards appear as choices within the "New Dialog", and are typically used to create local catalog entires for DataStores and Grid Coverage Exchanges.


  • point - a fully qualified identifier of the target extension point
  • id - an optional identifier of the extension instance
  • name - an optional name of the extension instance

<!ELEMENT wizard (description?)>

<!ATTLIST wizard

id               CDATA #REQUIRED

name             CDATA #REQUIRED

icon             CDATA #IMPLIED

class            CDATA #REQUIRED

helpHref         CDATA #IMPLIED

descriptionImage CDATA #IMPLIED>

  • id - a unique name that can be used to identify this wizard
  • name - a translatable name of the wizard that will be used in the dialog box
  • icon - a relative path of an icon that will be used together with the name to represent the wizard as one of the choices in the creation dialog box
  • class - a fully qualified name of the Java class implementing net.refractions.udig.registry.ui.IRegisterWizard
  • helpHref - a help url that can describe this wizard in detail
  • descriptionImage - a larger image that can help describe this wizard

<!ELEMENT description (#CDATA)>

an optional subelement whose body contains a short text describing what the wizard will do when started



Examples:
Following is an example of a register wizard configuration: 
<code>

   
<extension point=
"net.refractions.udig.registry.ui.registerWizards"
>
 
      
<wizard 
          id=
"net.refractions.wms.ui.GCEWizard"
 
          name=
"Grid Coverage Exchange"
 
          icon=
"./icons/WMSwizard1.gif"
 
          class=
"net.refractions.wms.ui.WMSWizard"
>
 
          
<description>
 
              Create a simple Web Map Server entry in the Grid Coverage Exchange
              category
          
</description>
 
      
</wizard>
 
   
</extension>


</code>

API Information:
The value of the class attribute must represent a class that implements net.refractions.udig.register.ui.IRegisterWizard. If the wizard is created from within the Register Wizard it will be inserted into the existing wizard. If the wizard is launched as a shortcut it will appear standalone as a separate dialog box.

Supplied Implementation:
The registry comes with wizards for creating resources of the following types: DataStore, GCE, WMS. These wizards are registered using the same mechanism as described above. Additional wizards may also appear, subject to particular platform installation.
